# 🥭 MangoBox

**กล่องวัดแก๊สเอทิลีน — พิสูจน์ว่าเอทิลีนเร่งการสุกของผลไม้**

ชุดเรียนรู้ระบบสมองกลฝังตัวสำหรับนักเรียนมัธยม ใช้ ESP32-S3 + เซ็นเซอร์แก๊ส/สี/อุณหภูมิ
สร้างได้ทั้ง **เครื่องวัดความสุกใช้จริง** และ **ชุดทดลองวิทยาศาสตร์**

> เวิร์กช็อป: คณะวิศวกรรมศาสตร์ มหาวิทยาลัยบูรพา × โรงเรียนพลูตาหลวงวิทยา
> วิทยากร: ผศ.ดร.สัญชัย เอียดปราบ (ESCE, Burapha Engineering)

## แนวคิดหลัก: แยก "เหตุ" ออกจาก "ผล"

| | วัดด้วย | คืออะไร |
|---|---|---|
| **เหตุ** (ตัวแปรต้น) | SGP30 → TVOC (ppb) | ระดับแก๊สเอทิลีนในกล่อง |
| **ผล** (ตัวแปรตาม) | TCS34725 → สีเปลี่ยน/วัน | อัตราการสุก |

**สมมติฐาน:** กล่องที่มีเอทิลีนสูงกว่า → มะม่วงจะสุกเร็วกว่า

### การทดลอง 4 กล่อง

| กล่อง | ใส่อะไร | คาดว่า |
|---|---|---|
| **A** | มะม่วง + กล้วยสุกงอม | เอทิลีนสูงสุด → สุกเร็วสุด |
| **B** | มะม่วงอย่างเดียว | control |
| **C** | มะม่วง + ด่างทับทิม (ดูดซับ) | เอทิลีนต่ำสุด → สุกช้าสุด |
| **D** | กล่องเปล่า | baseline ของเซ็นเซอร์ |

ถ้าได้ **TVOC: A > B > C** และ **อัตราการสุก: A > B > C** ไปทางเดียวกัน = พิสูจน์สำเร็จ

## สองโหมดในบอร์ดเดียว

| | **METER** (ใช้งานจริง) | **LAB** (ทำโครงงาน) |
|---|---|---|
| ตอบคำถาม | "มะม่วงลูกนี้กินได้หรือยัง?" | "เอทิลีนเร่งการสุกจริงไหม?" |
| การรวมค่า | รวมสี 60% + แก๊ส 40% | **ไม่รวม** — แยกเหตุ/ผลเด็ดขาด |
| จอแสดง | ความสุก 0–100% + ดิบ/พอดี/งอม | TVOC \| RIPE แยกช่อง + อัตราการสุก |

**สลับโหมด:** กดปุ่ม BOOT ค้าง 2 วินาที (ปุ่มมีบนบอร์ดแล้ว)
**บันทึกประจำวัน (LAB):** กด BOOT สั้น ๆ — เก็บลง NVS ถอดปลั๊กแล้วข้อมูลไม่หาย

> **จุดขาย:** ข้อมูลจากโหมด LAB ใช้ **คาลิเบรตเกณฑ์ของโหมด METER** ได้ — ทำวิทยาศาสตร์ก่อน แล้วเอาผลไปสร้างเครื่องมือ

## ฮาร์ดแวร์

| อุปกรณ์ | I2C Address | หน้าที่ |
|---|---|---|
| ESP32-S3 N16R8 (DevKitC-1) | — | ตัวประมวลผล (16MB Flash / 8MB PSRAM) |
| SGP30 | `0x58` | แก๊ส TVOC (ppb) + eCO2 |
| TCS34725 | `0x29` | สี (RGB → Hue) |
| BME280 | `0x76` / `0x77` | อุณหภูมิ / ความชื้น |
| OLED SSD1306 | `0x3C` / `0x3D` | จอแสดงผล |

### การต่อสาย

```
เซ็นเซอร์ทุกตัว (ต่อขนานบนบัส I2C เดียวกัน):
  VCC → 3V3      GND → GND
  SDA → GPIO8    SCL → GPIO9

พัดลม → GPIO10  (ผ่านทรานซิสเตอร์เท่านั้น! ห้ามต่อตรง)
ปุ่ม  → GPIO0   (ปุ่ม BOOT บนบอร์ด)
RGB   → GPIO48  (WS2812 บนบอร์ด)
```

⚠️ **ESP32-S3 ไม่มีขา GPIO22–25** และ GPIO26–37 ใช้กับ flash/PSRAM
📷 GPIO8/9/10 = ขากล้อง (CAM_Y4/Y3/Y5) → ถ้าจะต่อกล้องทำ AI ต้องย้าย I2C ไป GPIO1/2

## เริ่มต้นใช้งาน

### 1) ตั้งค่า Arduino IDE (สำคัญมาก!)

| Tools | ต้องเป็น |
|---|---|
| Board | ESP32S3 Dev Module |
| **USB CDC On Boot** | **Enabled** ← ไม่เปิด = Serial เงียบสนิท |
| Flash Size | 16MB (128Mb) |
| **PSRAM** | **OPI PSRAM** ← ไม่เปิด = เสีย 8MB ที่ซื้อมา |

### 2) ติดตั้งไลบรารี

ค้นใน **Library Manager** ด้วยชื่อเหล่านี้ (ชื่อไลบรารี ≠ ชื่อไฟล์ `.h`!):

- `Adafruit Unified Sensor` ← ตัวที่ให้ `Adafruit_Sensor.h`
- `Adafruit BME280 Library`
- `Adafruit SGP30 Sensor`
- `Adafruit TCS34725`
- `Adafruit SSD1306` + `Adafruit GFX Library`
- `Adafruit BusIO`

> **ทางลัด:** ลง BME280 ก่อน แล้วกด **"Install All"** ตอนถาม dependencies

### 3) ไล่ทีละขั้น

```
firmware/steps/step01_blink      →  บอร์ดไหว?
firmware/steps/step02_i2c_scan   →  เจอเซ็นเซอร์ครบไหม?
firmware/steps/step03_bme280     →  อ่านอุณหภูมิ
firmware/steps/step04_oled       →  แสดงบนจอ
firmware/steps/step05_color      →  สี → %ความสุก
firmware/steps/step06_gas    ★   →  เห็นแก๊สพุ่ง! (ห้ามข้าม)
firmware/steps/step07_fan        →  พัดลม + ทรานซิสเตอร์
firmware/steps/step08_meter  ★   →  รวมร่างเป็นเครื่องวัด
firmware/steps/step09_lab_mode   →  แยกเหตุ-ผล + บันทึกข้ามวัน
firmware/steps/step10_dashboard  →  ดูบนมือถือผ่าน WiFi
```

## 🔧 ไม่มีเซ็นเซอร์? ใช้ MOCK MODE

`firmware/steps/step00_mock` — **จำลองทั้งระบบโดยไม่ต้องมีเซ็นเซอร์เลย**

- จำลองมะม่วงค่อย ๆ สุก (20 วินาที = 1 วัน)
- ทดลอง 4 กล่อง A/B/C/D ได้จริง (เปลี่ยน `BOX_ID`)
- ทดสอบสูตรคำนวณ · โหมด · ปุ่ม · NVS · RGB ได้ครบ
- **ใช้เป็นตาข่ายกันตก** เมื่อเซ็นเซอร์กลุ่มไหนเสียหน้างาน

## 🐛 เครื่องมือไล่บั๊ก

| เครื่องมือ | ใช้เมื่อ |
|---|---|
| `firmware/tools/i2c_scanner` | หาว่าเจอเซ็นเซอร์อะไรบ้าง |
| `firmware/tools/i2c_stability` | **เซ็นเซอร์เจอบ้างไม่เจอบ้าง** — สแกน 20 รอบ ชี้ว่าตัวไหนสายหลวม |

### กับดักที่เจอจริง

1. **`USB CDC On Boot` ไม่เปิด** → Serial เงียบสนิท (เจอบ่อยสุด)
2. **RGB LED อยู่ GPIO48 ไม่ใช่ 38** → และต้องใช้ `neopixelWrite()` ไม่ใช่ `digitalWrite()`
3. **`SSD1306.begin()` โกหกได้!** → คืน `true` ทั้งที่ไม่ได้ต่อจอ ต้องเช็ก I2C ACK เอง:
   ```cpp
   bool i2cAlive(uint8_t addr) {
     Wire.beginTransmission(addr);
     return (Wire.endTransmission() == 0);
   }
   ```
4. **ชื่อไลบรารี ≠ ชื่อไฟล์ `.h`** → `Adafruit_Sensor.h` ต้องค้นว่า "Adafruit Unified Sensor"
5. **สายหลวม = ปัญหาอันดับ 1** → ใช้ `Wire.setClock(50000)` ช่วยได้เยอะ

> **บทเรียนวิศวกรรม:** *"อย่าเชื่อสิ่งที่คนอื่นเขียนมา — ตรวจสอบเอง"*

## ⚠️ ข้อจำกัด (ต้องเขียนในรายงาน)

- **SGP30 วัด "สารระเหยรวม (TVOC)" ไม่ใช่เอทิลีนบริสุทธิ์** — เป็นเพียงตัวแทน (พร็อกซี่)
- สิ่งที่เชื่อถือได้คือ **การเปรียบเทียบระหว่างกล่อง** ไม่ใช่ตัวเลขสัมบูรณ์
- **eCO2 เป็นค่าประมาณ** จากไฮโดรเจน ไม่ใช่ CO₂ ที่วัดจริง
- SGP30 ต้อง **burn-in** (เสียบไฟทิ้งไว้) ถึงจะให้ค่าที่แม่น

## 🚀 ต่อยอด: AI + กล้อง

บอร์ดนี้**มีช่องต่อกล้อง FPC มาให้แล้ว** → ทำ TinyML ได้โดยไม่ต้องซื้อบอร์ดใหม่

1. **เก็บชุดข้อมูล** — กล้องถ่ายรูป + เซ็นเซอร์บอกความสุก % ในวินาทีเดียวกัน
   → ได้ข้อมูล **ติดป้ายตัวเองอัตโนมัติ** (แก้จุดตายของโปรเจกต์ AI โรงเรียนทั่วไป)
2. **เทรนโมเดล** — Edge Impulse จำแนกภาพ ดิบ/พอดี/งอม
3. **ทำนายวันสุก** — รวมภาพ (เห็นตอนนี้) + แก๊ส (บอกอนาคต)

> กล้องเห็นสี/รอยช้ำ แต่ **ดมกลิ่นไม่ได้** — แก๊สจึงยังจำเป็น เพราะบอกอนาคตที่ตายังไม่เห็น
