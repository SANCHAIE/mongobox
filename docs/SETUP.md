# MangoBox — การตั้งค่าก่อนเวิร์กช็อป (สำหรับครูผู้ดูแล)

เตรียม "ล่วงหน้า" ให้เสร็จก่อนวันงาน จะประหยัดเวลาได้ 1–2 ชั่วโมง

## 1) ติดตั้ง Arduino IDE + บอร์ด ESP32
1. ติดตั้ง **Arduino IDE 2.x**
2. เปิด `File > Preferences` วางลิงก์นี้ในช่อง *Additional Boards Manager URLs*:
   `https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json`
3. `Tools > Board > Boards Manager` ค้น **esp32** โดย Espressif แล้ว Install
4. เลือกบอร์ด **ESP32S3 Dev Module** แล้วตั้งค่า (สำคัญมากสำหรับ S3):
   - **USB CDC On Boot : Enabled**  ← ถ้าไม่เปิด Serial Monitor จะไม่ขึ้นข้อความ
   - **Flash Size : 16MB (128Mb)**
   - **PSRAM : OPI PSRAM**
   - Upload Speed 921600, Baud (Serial) 115200
5. เสียบสาย **USB-C** เข้าพอร์ตของบอร์ด (บอร์ด N16R8 นี้ใช้ USB-C ต่อตรง ไม่ต้องลงไดรเวอร์แยกในเครื่องส่วนใหญ่)

## 2) ติดตั้งไลบรารี (Library Manager: `Tools > Manage Libraries`)
ค้นและติดตั้งทั้งหมดนี้ (กด Install dependencies เมื่อถูกถาม):

| ไลบรารี | ใช้กับ |
|---|---|
| **Adafruit SGP30 Sensor** | เซ็นเซอร์แก๊ส (TVOC / eCO2) |
| **Adafruit TCS34725** | เซ็นเซอร์สี |
| **Adafruit BME280 Library** | อุณหภูมิ/ความชื้น |
| **Adafruit Unified Sensor** | (dependency ของ BME280) |
| **Adafruit SSD1306** | จอ OLED |
| **Adafruit GFX Library** | (dependency ของ OLED) |
| **Adafruit BusIO** | (dependency ของ I2C) |

## 3) ทดสอบว่าเครื่องพร้อม
- เปิดไฟล์ `mango_ripeness_esp32/mango_ripeness_esp32.ino`
- ตั้ง `#define USE_WIFI 0` ไว้ก่อนตอนทดสอบครั้งแรก (เปิดเป็น 1 ทีหลัง)
- กด **Upload** ควรได้ข้อความ `[OK] ...พร้อม` ครบทุกตัวใน Serial Monitor (115200)

## 4) เช็ก I2C address (ถ้ามีตัวไหนหาไม่เจอ)
รันสเก็ตช์ **i2c_scanner** (ในโฟลเดอร์ `tools/`) จะได้เห็นว่ามี address ใดบ้างบนสาย
ค่าปกติ: `0x29` (สี) · `0x3C` (จอ) · `0x58` (แก๊ส SGP30) · `0x76` หรือ `0x77` (BME280)

> ถ้า BME280 เป็น `0x77` ไม่ต้องแก้โค้ด — เฟิร์มแวร์ลองทั้ง 0x76 และ 0x77 ให้อัตโนมัติแล้ว

---

## 5) สองโหมดในบอร์ดเดียว (ใช้ปุ่ม BOOT ที่มีบนบอร์ดอยู่แล้ว — ไม่ต้องต่อปุ่มเพิ่ม)

| | **โหมด METER** (ใช้งานจริง) | **โหมด LAB** (ทำโครงงาน) |
|---|---|---|
| ตอบคำถาม | "มะม่วงลูกนี้กินได้หรือยัง?" | "เอทิลีนเร่งการสุกจริงไหม?" |
| จอแสดง | คะแนนความสุกรวม 0–100% + ดิบ/พอดี/งอม | ETHYLENE \| RIPE แยกช่อง + อัตราการสุก |
| การรวมค่า | รวมสี 60% + แก๊ส 40% | **ไม่รวม** แยกเหตุ–ผลเด็ดขาด |
| ใช้ตอน | คัดผลไม้ ตรวจก่อนกิน/ก่อนส่ง | ทดลอง 4 กล่อง เก็บข้อมูลข้ามวัน |

### วิธีใช้ปุ่ม BOOT
- **กดค้าง 2 วินาที** = สลับโหมด (จอจะขึ้น SWITCH MODE)
- **กดสั้น ๆ 1 ครั้ง** (เฉพาะโหมด LAB) = บันทึกผลประจำวัน — ทำวันละครั้ง เวลาเดิม
- โหมดที่เลือกไว้จะถูกจำไว้ ถอดปลั๊กแล้วเปิดใหม่ยังอยู่โหมดเดิม

> **ห้ามกดปุ่ม BOOT ค้างไว้ตอนกด RESET หรือตอนเสียบ USB** เพราะจะเข้าโหมดอัปโหลด บอร์ดจะไม่รันโปรแกรม
> (กดตอนโปรแกรมทำงานอยู่แล้ว ปลอดภัย ไม่มีปัญหา)

### คำสั่งทาง Serial Monitor
| พิมพ์ | ทำอะไร |
|---|---|
| `m` | สลับโหมด |
| `p` | พิมพ์ตารางบันทึกประจำวันทั้งหมด |
| `r` | ล้างข้อมูลบันทึก (โหมดยังคงเดิม) |

## 6) ข้อมูล CSV — ได้ครบทุกคอลัมน์เสมอ ไม่ว่าอยู่โหมดไหน
```
millis,Mode,Box,Temp,Hum,Ethylene,Hue,ColorRipe,MeterScore,RatePerDay
```
โหมดเป็นเพียง "ชั้นแสดงผล" เท่านั้น — ข้อมูลดิบไม่เสียหาย นำไปวิเคราะห์ได้ทั้งสองแนวทาง

## 7) ตำแหน่งวางบอร์ด (สำคัญ)
วางบอร์ด **ไว้นอกกล่อง** แล้วเดินสายเซ็นเซอร์เข้าไปในกล่อง เพราะ
1. ความชื้นในกล่องปิดสูงมาก ไม่ควรให้บอร์ดโดน
2. ต้องกดปุ่ม BOOT ได้โดยไม่ต้องเปิดฝากล่อง (เปิดฝา = แก๊สหนีหมด)

## 8) คาลิเบรตเกณฑ์ (ทำหลังเก็บข้อมูลจริง)
ค่าเหล่านี้อยู่ต้นไฟล์ `.ino` ปรับได้เมื่อมีข้อมูลจริงจากโหมด LAB:
```cpp
#define HUE_RAW    120.0   // สีเขียว (ดิบ)  = 0%
#define HUE_RIPE    55.0   // สีเหลือง (สุก) = 100%
#define TVOC_BASE    0.0   // TVOC อากาศสะอาด (ppb)
#define TVOC_HIGH  500.0   // TVOC ตอนผลไม้สุกจัด (ppb)  <-- ปรับตามจริง
```
> นี่คือจุดเชื่อมของสองโหมด: **เอาข้อมูลที่เก็บได้จากโหมด LAB มาปรับเกณฑ์ให้โหมด METER แม่นขึ้น**

## 9) เรื่องที่ต้องรู้เกี่ยวกับ SGP30 (สำคัญมาก — ต่างจาก SGP40)

| | SGP30 (ที่เราใช้) |
|---|---|
| I2C address | **0x58** |
| ไลบรารี | **Adafruit SGP30** |
| ค่าที่ได้ | **TVOC (ppb)** + eCO2 (ppm) |
| ต้องอุ่นเครื่อง | 15 วินาทีแรกคืนค่า 0 ppb / 400 ppm เสมอ (ปกติ) |
| Baseline | **ต้องมี** ถึงจะแม่น — โปรแกรมนี้ save/restore ให้อัตโนมัติ |

### Baseline คืออะไร ทำไมสำคัญ
SGP30 ต้องรู้ว่า "อากาศสะอาด" ในบริเวณนี้หน้าตาเป็นยังไง ถึงจะบอก TVOC ได้แม่น
- โปรแกรมจะ **เก็บ baseline ลง NVS ทุก 1 ชั่วโมง** และ **คืนค่าให้อัตโนมัติตอนเปิดเครื่อง**
- **ครั้งแรกสุด: ควรเสียบไฟทิ้งไว้ประมาณ 12 ชั่วโมง (burn-in)** ในที่อากาศถ่ายเท ก่อนใช้งานจริง
- ถ้ายังไม่ได้ burn-in ค่าจะยังไม่นิ่ง — **ยังเห็น "แนวโน้ม" ได้ แต่ตัวเลขสัมบูรณ์เชื่อไม่ได้เต็มที่**
- พิมพ์ `b` ใน Serial Monitor = สั่งเก็บ baseline ทันที

> **คำแนะนำ:** เสียบบอร์ดทิ้งไว้ตั้งแต่คืนก่อนเวิร์กช็อป จะได้ค่าที่แม่นตั้งแต่เช้า

### eCO2 — ห้ามเคลมเกิน
eCO2 เป็นค่า **"ประมาณ"** ที่คำนวณจากไฮโดรเจน **ไม่ใช่ CO2 จริง** ที่วัดโดยตรง
ถ้าจะใส่ในรายงาน ต้องเขียนกำกับว่าเป็น *equivalent* CO2 ไม่ใช่ค่าที่วัดได้จริง

# 🔧 บทเรียนจากการทดสอบจริง (อ่านก่อนสอน!)

ส่วนนี้คือปัญหาที่ **เจอจริงตอนทดสอบ** ไม่ใช่ทฤษฎี — เตรียมรับมือไว้ก่อน

## 1) ตั้งค่า Arduino IDE — 4 อย่างที่ห้ามพลาด

| Tools | ต้องเป็น | ถ้าตั้งผิดจะเกิดอะไร |
|---|---|---|
| Board | ESP32S3 Dev Module | คอมไพล์ไม่ผ่าน |
| **USB CDC On Boot** | **Enabled** | **Serial Monitor เงียบสนิท** (เจอบ่อยสุด) |
| Flash Size | 16MB (128Mb) | พื้นที่ไม่พอ |
| **PSRAM** | **OPI PSRAM** | PSRAM = 0 KB → เสียของ 8MB ที่ซื้อมา + ต่อยอด AI ไม่ได้ |

> ถ้าตั้ง OPI PSRAM แล้วยังได้ 0 KB → ลอง **QSPI PSRAM** (บอร์ดโคลนบางล็อตใช้แบบนี้)

## 2) RGB LED อยู่ที่ GPIO48 (ไม่ใช่ 38!)

บอร์ดนี้**ไม่มี LED ธรรมดา** มีแต่ RGB (WS2812)
- พินเอาต์ทางการของ Espressif บอก **38** แต่บอร์ดโคลนตัวนี้ใช้ **48**
- `digitalWrite()` ใช้ไม่ได้ ต้องใช้ **`neopixelWrite(48, r, g, b)`**
- **ความสว่างใช้ 10 พอ** (ค่า 60 แยงตามาก)

## 3) ⚠️ อย่าเชื่อ `begin()` ของไลบรารี — มันโกหกได้!

**เรื่องจริงที่เจอ:** `Adafruit_SSD1306.begin()` คืนค่า `true` **ทั้งที่ไม่ได้ต่อจอเลย**

เพราะมันไม่เช็ก I2C ACK จริงจัง แค่ยิงคำสั่ง init ออกไปแล้ว return true

**วิธีที่ถูก — เช็ก ACK เองเสมอ:**
```cpp
bool i2cAlive(uint8_t addr) {
  Wire.beginTransmission(addr);
  return (Wire.endTransmission() == 0);   // 0 = มีคนตอบจริง
}

// ใช้แบบนี้:
if (i2cAlive(0x3C)) { oled.begin(SSD1306_SWITCHCAPVCC, 0x3C); }
else Serial.println("[X] ไม่พบจอ");
```

> **นี่คือบทเรียนวิศวกรรมที่ดีมากสำหรับนักเรียน:** *"อย่าเชื่อสิ่งที่คนอื่นเขียนมา — ตรวจสอบเอง"*
> ถ้าไม่ทำแบบนี้ เด็กจะเจอ "โปรแกรมบอก OK แต่จอดำ" แล้วหาสาเหตุไม่เจอเป็นชั่วโมง

## 4) ⚠️ ชื่อไลบรารี ≠ ชื่อไฟล์ header

| `#include` | ต้องค้นใน Library Manager ว่า |
|---|---|
| `Adafruit_Sensor.h` | **Adafruit Unified Sensor** ← กับดัก! |
| `Adafruit_BME280.h` | Adafruit BME280 Library |
| `Adafruit_SGP30.h` | **Adafruit SGP30 Sensor** (ไม่ใช่ SGP40!) |
| `Adafruit_TCS34725.h` | Adafruit TCS34725 |
| `Adafruit_SSD1306.h` | Adafruit SSD1306 |
| `Adafruit_GFX.h` | Adafruit GFX Library |

**ทางลัด:** ลง *Adafruit BME280 Library* ก่อน แล้วกด **"Install All"** ตอนถาม dependencies → ได้ Unified Sensor + BusIO มาให้อัตโนมัติ

## 5) สายหลวม = ปัญหาอันดับ 1

**อาการ:** สแกน I2C แล้ว "เจอบ้าง ไม่เจอบ้าง" สลับไปมา

**วิธีตรวจ:** รัน `tools/i2c_stability` — สแกน 20 รอบ ถ้าตัวไหนไม่ได้ 20/20 = สายหลวม

**วิธีแก้:**
- ใช้ **`Wire.setClock(50000)`** ลดบัสเป็น 50kHz — ทนสายยาว/หลวมดีขึ้นมาก (ใส่ไว้ในทุก STEP แล้ว)
- ลดจำนวนจุดต่อ (สายจัมเปอร์ต่อกันหลายทอด = จุดหลวมหลายจุด)
- ใช้สายสั้นที่สุด กดให้แน่น

## 6) I2C address ที่อาจไม่ตรงตามคาด

| อุปกรณ์ | ปกติ | แต่บางตัวเป็น |
|---|---|---|
| OLED | 0x3C | **0x3D** |
| BME280 | 0x76 | 0x77 |
| SGP30 | 0x58 | (คงที่) |
| TCS34725 | 0x29 | (คงที่) |

**SGP30 บางโมดูลต้องการไฟ 5V ไม่ใช่ 3V3** — ถ้าหาไม่เจอเลย ลองย้าย VCC ไป 5V

## 7) 📷 ขา I2C ชนกับกล้อง (เรื่องอนาคต)

บอร์ดนี้**มีช่องต่อกล้อง FPC มาให้แล้ว** (ดีมาก! ต่อยอด AI ได้โดยไม่ต้องซื้อบอร์ดใหม่)

**แต่:** GPIO8/9/10 ที่เราใช้ = **ขากล้อง (CAM_Y4/Y3/Y5)**

- ตอนนี้ไม่มีปัญหา เพราะยังไม่เสียบกล้อง
- **วันที่ต่อกล้องทำ AI → ต้องย้าย I2C ไปขาอื่น** เช่น GPIO1/GPIO2 (ว่างอยู่)

## 8) ตาข่ายกันตก: MOCK MODE

ถ้ากลุ่มไหนเซ็นเซอร์เสีย/ต่อไม่ติด → แฟลช `steps/step00_mock` แล้วเรียนต่อได้ทันที
จำลองมะม่วงสุกเต็มระบบ (แถมทดลอง 4 กล่องได้ใน 2 นาที!) ไม่มีใครหลุดจากกิจกรรม
